Config Tab Help

Target-level identity, scheduling, and mute controls.
Help Index Engine Doc

What This Tab Controls

  • TARGET_NAME: immutable identity key used in filenames and runtime result rows.
  • DESCRIPTION, TAG_LIST, DOCUMENT_URL, DASHBOARD_URL: operator context and links.
  • IS_ACTIVE: when false, scheduler still ticks but runtime inserts state=skipped.
  • IS_MUTED: action execution is suppressed and saved as TARGET_MUTED-NO_ACTION_TAKEN.
  • SCHEDULE_CRON: minute-based schedule gate checked by engine scheduler.

Scheduler Behavior

The scheduler loop evaluates every minute. For each target, it checks SCHEDULE_CRON and computes mute state for the next minute. This comes from both static mute flags and dynamic mute rules.

  • MUTE_BETWEEN_ENABLED + MUTE_BETWEEN_RULES: supports multiple day/time windows.
  • MUTE_UNTIL_ENABLED + MUTE_UNTIL: fixed datetime boundary; if now is before boundary, target is muted.
  • Cross-midnight windows are supported: 23:00-06:00 means late night into next day.
  • When a target is muted, evaluation still writes rows; only actions are skipped.

Mute Rules Structure

MUTE_BETWEEN_RULES is a JSON array. 

[
  {"start":"23:00","end":"06:00","days":["MON","TUE","WED","THU","FRI"]},
  {"start":"00:00","end":"23:59","days":["SAT","SUN"]}
]
  • start/end format is HH:MM.
  • days values are MON..SUN.
  • If days are missing in a rule, runtime defaults to all days.
