History Tab Help

Runtime intervals, raw execution records, import, and replay actions.
Help Index Engine Doc

Status History Table

The history table shows contiguous severity/state intervals per metric, not individual raw execution rows. Runtime rows are grouped when severity or state does not change.

  • started_at: first row in interval.
  • ended_at: last row in interval (current active interval has no end time).
  • last_message, last_action: latest values in that interval.

Metric Filter

  • Dropdown filters interval view by metric name.
  • All metrics shows all intervals for the target.
  • Filter selection also drives behavior in Run Normal Action popup.

History Actions

  • View Raw Instance: loads raw anbu_result rows on demand (up to limit).
  • Delete History: removes all runtime rows for the target.
  • Import History: executes read-only history SQL (.hql) and inserts rows into runtime table.
  • Refresh: reloads latest intervals and states.

Run Normal Action

  • Uses selected metric from dropdown. If dropdown is All metrics, it processes all configured metrics.
  • Only metrics whose latest severity is not NORMAL are executed.
  • Normal action script is called with severity forced to 1 and the provided message.
  • Popup shows progress bar while request runs and outputs combined stdout/stderr per metric.
