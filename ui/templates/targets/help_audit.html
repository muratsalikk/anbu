What Is Stored

Each audited change stores a snapshot of target files and metadata:

  • .env content
  • .sql content
  • .hql content (when present)
  • edited_at and edited_by

Actions in Audit Tab

  • View: opens snapshot content in a separate view with env/sql/hql tabs.
  • View Changes: shows diff against previous snapshot.
  • Restore: writes snapshot back to files after confirmation.
  • The latest snapshot's restore is disabled in edit page because it already matches current state.

Restore Semantics

  • Restore replaces live target files on disk.
  • Restore itself is audited as a new snapshot for traceability.
  • If .hql in snapshot is empty, current history SQL file is removed.
  • Always verify target behavior after restore using History and latest runtime rows.
